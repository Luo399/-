#  后端开发执行工作流（仅执行以下任务，严禁越权）
## 任务1：初始化后端服务（仅项目开头执行一次）
1.  仅在backend/目录生成标准结构：app.js（服务入口）、config.js（配置文件）、db.js（数据库配置）、routes/（路由目录）、controllers/（业务逻辑目录），不得遗漏、不得新增多余文件
2.  必须配置CORS跨域、统一响应中间件、全局错误处理中间件，确保适配前端域名（GitHub Pages或自定义域名）
3.  必须集成日志、缓存、文件上传功能，分别对应logs/、cache/、uploads/目录，确保电脑访问时功能可正常使用
4.  必须初始化SQLite数据库，创建用户认证（注册、登录）、基础数据CRUD相关表及接口，接口符合统一规范
5.  必须配置环境变量适配，生成云平台部署指令（npm install && npm run dev），明确部署步骤

## 任务2：接口开发/优化（按需执行）
1.  仅在backend/routes/、backend/controllers/目录开发、修改接口，不得触碰其他目录
2.  新增接口必须添加合理缓存策略，优化电脑访问速度，不得出现无缓存、缓存异常情况
3.  必须完善日志记录，包含访问IP、访问时间、请求路径、响应状态、错误信息，便于电脑访问问题排查
4.  开发的接口必须兼容云平台部署，无任何本地依赖，确保部署后电脑端可正常调用
5.  接口开发必须严格遵循统一响应格式、路由格式、状态码规范，不得有任何偏差

## 任务3：部署适配优化（按需执行）
1.  检查backend/目录下所有代码，确保可直接上传GitHub、直接部署至云平台，无任何部署障碍
2.  优化缓存配置，适配CDN加速，确保电脑访问流畅，无卡顿、加载缓慢问题
3.  检查日志、上传文件功能，确保可在云平台持久化存储，电脑上传、访问数据正常
4.  检查跨域配置，确保电脑端前端（GitHub Pages或自定义域名）可正常调用所有接口，无跨域报错

## 绝对禁止执行（触碰即终止任务）
1.  禁止任何修改、删除、重命名index.html、public/目录及其中所有文件的操作
2.  禁止在backend/目录以外的任何位置新增、修改、删除后端相关代码或文件
3.  禁止将.env文件中的私密配置（密钥、数据库地址等）写入代码、提交至GitHub
4.  禁止添加任何本地依赖，禁止出现无法在云平台部署的代码
5.  禁止删除、修改logs/、cache/、uploads/、temp/目录及其中的自动生成逻辑
6.  禁止偏离任务要求，禁止执行任务以外的操作（如优化前端、修改目录结构等）
7.  禁止开发不符合接口规范、响应格式、路由格式的接口
8.  禁止修改数据库表结构、删除数据库数据（除非明确指令）
