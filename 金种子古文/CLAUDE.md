# AI 后端开发强制指令

## 核心任务指令

1. 仅负责后端开发，通过RESTful API与前端通信，不参与任何前端相关操作
2. 所有后端代码必须存放于backend/目录，严格遵循目录规范
3. 开发内容需适配电脑访问，兼容云平台（Railway/Render/Cyclic）部署和GitHub托管
4. 所有私密配置（密钥、数据库地址、端口）必须仅存于.env文件，不得写入代码、不得提交至GitHub，确保数据安全

## 技术栈强制要求

- 必须使用Node.js + Express + SQLite（或云平台提供的数据库）
- 必须开启CORS跨域，服务端口严格遵循.env环境变量配置
- 必须集成日志、缓存、文件上传持久化功能，确保电脑访问稳定、安全

## 目录执行标准

- 后端代码：仅在backend/目录开发、修改，不得在其他目录新增/修改后端相关文件
- 日志目录：logs/（自动生成，不得手动删除/修改，不上传GitHub）
- 缓存目录：cache/（自动生成，不得手动删除/修改，不上传GitHub）
- 临时文件目录：temp/（自动生成，不得手动删除/修改，不上传GitHub）
- 上传文件目录：uploads/（自动生成，确保电脑上传文件正常存储，不上传GitHub）
- 前端相关：public/目录、index.html文件，不得有任何修改、删除、重命名操作\
 - "金种子古文"项目结构如有更改，写入project\_struct.md文件内，标明时间

## 接口执行标准

1. 响应格式必须统一：{ success: boolean, data: any, message: string }，不得有任何偏差
2. 路由格式必须统一：/api/module/action，不得自定义路由格式
3. 状态码必须规范：200=成功、400=参数错误、401=权限不足、500=服务异常
4. 所有接口必须添加合理缓存策略，适配CDN，提升电脑访问速度

## 部署适配指令

1. 开发的后端代码必须可直接上传GitHub，兼容.gitignore配置，不携带非必要文件
2. 代码必须可直接部署至云平台，无需修改任何代码即可运行
3. 日志、缓存、上传文件必须适配云平台持久化存储，确保电脑上传、访问数据正常
4. 跨域配置必须兼容前端GitHub Pages域名或自定义域名，确保电脑端前端可正常调用接口

## 绝对红线（严禁触碰）

1. 严禁修改、删除、重命名index.html、public/目录及其中所有文件
2. 严禁将后端代码存放于backend/目录以外的任何位置
3. 严禁将.env文件中的私密配置写入代码、提交至GitHub
4. 严禁添加本地依赖，确保代码可直接在云平台部署运行
5. 严禁修改、删除logs/、cache/、uploads/、temp/目录及自动生成逻辑
6. 严禁偏离接口规范、响应格式、路由格式，不得随意更改技术栈、数据库、端口等配置
